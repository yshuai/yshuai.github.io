ubuntu+github+hexo搭建个人博客

为什么选择github+hexo？

  1. 免费
  2. 主题够用。不用自己制作。而且自己可以自定义颜色，字体等等
  3. 建站简单。
  4. 网上有很多github的教程。

总之，如果你有时间，有耐心，想建个博客耍耍，又不想花钱。github+hexo很适合。

###GitHub 与 静态博客
静态博客是个新生事物。静态博客的流行跟著名的开源社区Github的支持是分不开的。因此，绝大部分静态博客平台设计得还是比较极客，更适合有一定编程技能的人来使用，并不适合完全零起点的完全没有编程知识的普通用户。而Github除提供在线Markdown编辑器之外，还提供了GithubPage服务，可以将用户托管在Github上的Markdown博客发布为静态网站。博主完全可以将自己的博客站免费寄放在Github上。此外，通过Github发布博客，还有一个额外的好处，那就是你的博客文章可以用Git来管理，这样你就可以在Github上获得博客文章的完整发布历史（像程序员管理源代码一样），并且可以随时获取历史版本或回退和回退到特定的版本上，相当于有了强大的备份系统。

Hexo
hexo出自台湾大学生tommy351之手，是一个基于Node.js的静态博客程序，其编译上百篇文字只需要几秒。hexo生成的静态网页可以直接放到GitHub Pages，BAE，SAE等平台上。

##安装篇

###安装Node.JS

有许多安装Node.JS的不同的方法，我们可以选择其一。通过本篇文章我们将手把手带着你在Ubuntu 15.04（我的ubuntu15.10也成功了，附上原链接：安装node.js）上安装Node.Js，在此之前请卸载旧版本的包以免发生包冲突。

从源代码安装Node.JS

让我们开始从源代码安装Node.JS之前，请确认系统上的所有的依赖包都已经更新到最新版本。然后跟着以下步骤来开始安装：

步骤1: 升级系统

用以下命令来升级系统，并且安装一些Node.JS必要的包。

  1. root@ubuntu-15:~# apt-get update
  2. root@ubuntu-15:~# apt-get install python gcc make g++

步骤2: 获取Node.JS的源代码

安装好依赖包之后我们可以从官方网站上下载Node.JS的源代码。node.js官网地址下载Source Code（源码）

步骤3: 开始安装

现在我们进入源代码的目录，然后运行.configure文件

  1. root@ubuntu-15:~# ls
  2. node-v0.12.4 node-v0.12.4.tar.gz
  3. root@ubuntu-15:~# cd node-v0.12.4/
  4. root@ubuntu-15:~/node-v0.12.4# ./configure
  5. #这一步一定要以root权限。root@ubuntu-15:~/node-v0.12.4# sudo make install(非root用户要sudo)

安装后测试

只要运行一下上面的命令就顺利安装好了Node.JS，现在我们来确认一下版本信息和测试以下Node.JS是否可以运行输出。

  1. root@ubuntu-15:~/node-v0.12.4# node -v

查看是否输出版本信息即可（如果失败请自行百度谷歌。上文中的链接里也有安装node.js的其他方法）

Ubuntu安装Git

在ubuntu中安装git还是相当简单的,直接使用包管理器进行安装就好:

  1. $sudo apt-get install git
  2. $sudo apt-get install git-core

Nodejs 和 git都已经安装好了,接下来就可安装我们的主角Hexo了（如果普通权限下失败。可以试试root权限安装）

  1. $ npm install -g hexo-cli
    hexo部署失败

hexo deploy命令显示

  1. ERROR Deployer not found : github或者git时

deploy的type改成git，然后运行下npm install hexo-deployer-git –save
再hexo g
hexo d

Github pages 仓库创建

首先要注册下github，自己百度一下吧。
一定要验证邮箱。不然生成页面后会一直404。

点右下角的绿色按钮（New repository）
创建仓库名的格式是username.github.io
例如上图中的yshuai.github.io

勾选下边的那个Initialize this repository with a README（勾不勾选好像没什么区别。只是创建的仓库下多一个readme文件。根据个人情况而定）然后creat repository,即创建成功。

选择上图的setting，在跳转到的页面 Update your site 对应处点击“Automatic page generator”按钮，这样就有了一个github自动生成的页面用来测试的时候使用。之后点击继续。

之后，来到选择主题界面，选择主题并发布。
创建好这样的仓库后就可以在浏览器地址栏输入[username].github.io进行访问了.

搭建篇

开始搭建

初始化本地博客目录

  1. #在home新建一个文件夹,用于本地存放博客
  2. $mkdir Blogs
  3. $cd Blogs
  4. #用hexo命令在Blogs目录中初始化环境
  5. $hexo init
  6. $npm install

这个时候,其实Hexo为我们生成了一个默认的名为HelloWorld的一篇博客,接下来我们看下这篇博客

（一些和建站无关但是和以后有关的东西：
 Blogs内的文件
.
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
|   ├── _drafts
|   └── _posts
└── themes）

_config.yml文件整个博客的配置文件,
source/_posts文件夹包含了当前整个博客的所有文章
themes 文件夹包含了当前的主题文件.

生成静态文件

  1. #使用hexo generate命令可以将我们编写的markdown文件转换成html文件
  2. $hexo generate

这个时候在浏览器地址栏输入localhost:4000就可看到Hexo为我们生成的这边博客了
大概就是这个样子
本地服务开启

  1. wyq@wyq-m81 ~/Blog $ hexo server

  2. INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

    还有三步。博客便完成了。

  1. 更改主题。
  2. 上传至github。
  3. 发表文章。

可在此处下载hexo的主题：hexo主题下载，选择主题右侧的Demo可以访问安装了此主题的博客。点击主题名称。复制弹出页面的链接。

格式是git clone + 粘贴主题下载地址的链接 themes/主题名称

git clone https://github.com/coneycode/coney themes/coney

*在Blogs文件夹。更改_config.yml文件,在最后加入使用主题语句

进入主题目录。编辑主题文件

$cd themes/yilia

生成静态文件,在本地预览

$hexo generate

$hexo server

上传至github page

配置你的git只需要配置username和email就好了。（好像是这样，时间太久记不清了。）

在Blogs文件夹下_config.yml中,有个deployment一节,在这边我们进行github pages 的配置。

1. # Deployment
2. ## Docs: http://hexo.io/docs/deployment.html
3. deploy:
4.   type: git #注意,这边是git不是github,hexo3.0以后就要求使用git,不然会出现deploy失败情况.
5.   repo: https://github.com/yshuai/yshuai.github.io
6.   branch: master

tips：type，repo,branch是参数，前边有两个空格

配置好了后。输入一下命令上传：

  1. hexo clean
  2. hexo generate

  3. hexo deploy

    之后,在地址栏输入[username.github.io] 就可以访问你的博客了.

使用hexo new [postname]新建一篇博客
$hexo new firstBLog

完成之后,进入到source/_psots目录就可以看到我们刚才新建的博客了

$cd source/_posts
$ll
drwxr-xr-x 2 wyq wyq 4096 Jul 30 11:25 .
drwxr-xr-x 3 wyq wyq 4096 Jul 29 16:17 ..
-rw-r--r-- 1 wyq wyq   53 Jul 30 11:25 firstBlog.md

firstBlog.md就是我们刚才新建的一篇博客

博客文章都是md格式的markdown文件,我们打开来看下

1.#Hexo默认为我们生成了三行
2.title: firstBlog #这边博客的标题
3.date: 2015-07-30 11:25:44 #博客创建时间
4.tags: #博客的tag

那我们只需要使用markdown语法在这个文件里面写我们的博客就ok了,这里推荐在线markdwon编辑器,CMD markdown,至于markdown语法,我们只需要了解常用的就行了,学习成本几乎等于0,Github markdown语法
